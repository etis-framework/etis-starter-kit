# Standard Stage Directory Pattern

Use this directory pattern for ES-101 through ES-114.

```text
engineering/
  ES-XXX/
    README.md
    navigation.md
    engineering_context.md
    activities.md
    evidence.md
    outputs.md
    readiness_gate.md
    stage_manifest.md
    assets/
```

## File Purposes

| File | Purpose |
|---|---|
| README.md | Stage overview and orientation |
| navigation.md | Stage navigation and page order |
| engineering_context.md | Why the stage matters and what question it answers |
| activities.md | Guided engineering work |
| evidence.md | Evidence expectations and repository locations |
| outputs.md | Deliverables produced by the stage |
| readiness_gate.md | Completion and transition criteria |
| stage_manifest.md | Structured stage contract |
| assets/ | Stage-specific diagrams or supporting files |

## ES-100 Exception

ES-100 is a platform onboarding stage and has a custom structure:

```text
README.md
navigation.md
first_day_guide.md
engineering_journey.md
engineering_principles.md
stage_map.md
glossary.md
readiness_gate.md
stage_manifest.md
assets/
```

## Rule

Do not create stage-specific file names unless the stage genuinely requires them.
