# Stage Implementation Standard

This standard governs implementation of Engineering Stages ES-100 through ES-114.

## Core Requirement

Every Engineering Stage must feel like part of the same engineering platform.

Consistency is a product feature.

## Required Stage Files

Unless a stage has a documented reason to differ, use:

```text
README.md
engineering_context.md
activities.md
evidence.md
outputs.md
readiness_gate.md
stage_manifest.md
navigation.md
assets/
```

ES-100 is an exception because it provides platform onboarding and includes additional orientation pages.

## Standard Page Order

Each stage should present information in this order:

1. Overview
2. Purpose
3. Learning or engineering objectives
4. Engineering context
5. Inputs
6. Activities
7. Evidence
8. Outputs
9. Readiness gate
10. Next stage

## Required Page Behavior

Every page must answer:

- Where am I?
- Why am I here?
- What am I doing?
- What should I produce?
- How do I know I am done?
- What comes next?

## Engineering Stage Contract

Every stage must define:

- identity;
- purpose;
- inputs;
- activities;
- outputs;
- evidence;
- exit criteria;
- navigation;
- mentoring guidance.

## Quality Dimensions

Before freezing, every stage must pass:

- engineering review;
- cognitive review;
- navigation review;
- evidence review;
- publication review.

## Prohibited Content

Do not include:

- TODO markers;
- placeholder text;
- draft disclaimers;
- unexplained acronyms;
- dead-end pages;
- duplicate doctrine when a shared reference exists;
- AI-generated claims without verification guidance.
