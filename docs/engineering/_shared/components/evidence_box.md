# Shared Component — Evidence Box

Use an evidence box when a page or activity produces durable engineering evidence.

```markdown
📋 **Evidence Produced**

This activity should leave evidence in:

`docs/<area>/<artifact_name>.md`

The evidence should capture:

- decision made;
- rationale;
- assumptions;
- tradeoffs;
- review status;
- downstream impact.
```

## Purpose

The evidence box reminds engineers that engineering work should leave reviewable traces.

## When to Use

Use when the activity produces:

- a decision;
- a review record;
- a requirement;
- an architecture record;
- a risk analysis;
- a test result;
- a release record;
- an operational record;
- an AI-use record.

## When Not to Use

Do not use evidence boxes for purely explanatory content.

ES-100, for example, produces readiness rather than project-specific artifacts.
