# Shared Component — Readiness Gate Pattern

Every Engineering Stage ends with a readiness gate.

A readiness gate verifies whether the engineer or team is ready to advance.

## Standard Structure

```markdown
# ES-XXX Readiness Gate

## Purpose

This readiness gate verifies that the stage is complete and the next stage can begin responsibly.

## Completion Standard

You are ready to continue when each gate is satisfied.

## Gate 1 — Understanding

Question...

**Status:** ☐ Yes ☐ Needs review

## Gate 2 — Evidence

Question...

**Status:** ☐ Yes ☐ Needs review

## Gate 3 — Outputs

Question...

**Status:** ☐ Yes ☐ Needs review

## Gate 4 — Review

Question...

**Status:** ☐ Yes ☐ Needs review

## Exit Criteria

The stage is complete when...

## Continue

Continue to [Next Stage](../ES-XXX/README.md).
```

## Gate Categories

Use these categories as appropriate:

- understanding;
- inputs;
- activities;
- outputs;
- evidence;
- review;
- traceability;
- AI verification;
- stakeholder alignment;
- operational readiness;
- governance readiness;
- next-stage readiness.

## Rules

- Readiness gates are not quizzes.
- Readiness gates verify engineering readiness.
- Do not create artificial gates.
- Do not allow a stage to complete without evidence where evidence is required.
- Gates should be practical enough for individual, classroom, and team use.
