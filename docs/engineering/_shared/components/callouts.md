# Shared Component — Callout Library

The ETIS Engineering Platform uses a consistent callout vocabulary.

Use these callouts to reduce cognitive load and create a recognizable platform voice.

## Engineering Insight

```markdown
💡 **Engineering Insight**

Brief explanation of an engineering idea that improves judgment.
```

Use when the engineer should understand a deeper principle.

## Common Pitfall

```markdown
⚠️ **Common Pitfall**

Brief warning about a common error, misconception, or shortcut.
```

Use when preventing a likely downstream defect.

## AI Assistance

```markdown
🤖 **AI Assistance**

Guidance on appropriate and inappropriate AI use for this stage.
```

Use when AI can help but verification responsibility must remain clear.

## Repository Artifact

```markdown
📁 **Repository Artifact**

Expected location: `docs/.../artifact_name.md`
```

Use when a stage produces or references a durable repository artifact.

## Evidence Produced

```markdown
📋 **Evidence Produced**

This activity produces evidence that supports later engineering decisions.
```

Use when the engineer needs to recognize that work should leave a trace.

## Next Step

```markdown
➡️ **Next Step**

Continue to the next action.
```

Use sparingly. Prefer normal page-ending navigation where possible.

## Rules

- Do not overuse callouts.
- Use callouts to clarify, warn, or reinforce judgment.
- Do not use callouts as decoration.
- Keep callouts concise.
