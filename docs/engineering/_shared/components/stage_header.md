# Shared Component — Engineering Stage Header

Use this header at the beginning of every Engineering Stage page.

```markdown
**Engineering Stage:** ES-XXX — Stage Name  
**Current Page:** Page Name  
**Previous:** [Previous Page](previous.md)  
**Next:** [Next Page](next.md)
```

For the main stage README, use the expanded version:

```markdown
**Engineering Stage:** ES-XXX  
**Stage Name:** Stage Name  
**Purpose:** One-sentence purpose.  
**Estimated Effort:** 30–60 minutes | 1–2 hours | Half day | Full day | Multiple iterations  
**Prerequisites:** Prior stage or none  
**Produces:** Primary output  
**Next Stage:** [ES-XXX — Next Stage](../ES-XXX/README.md)
```

## Purpose

The stage header answers the engineer's first orientation questions:

- Where am I?
- What stage am I in?
- What page am I reading?
- What came before?
- What comes next?

## Rules

- Use the same format across all stages.
- Do not omit previous or next navigation unless the page is the first or final page of the entire platform.
- Keep the header compact.
- Do not duplicate the full stage manifest in the header.
