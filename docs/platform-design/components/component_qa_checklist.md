# Component QA Checklist

Use this checklist before accepting a component into the design system.

## Purpose

- [ ] The component has a clear purpose.
- [ ] The component reduces cognitive load.
- [ ] The component supports engineering work.
- [ ] The component is not merely decorative.

## Reusability

- [ ] The component can be used on more than one page.
- [ ] The component does not depend on one page's content.
- [ ] The component has clear required and optional fields.

## Content

- [ ] The component uses concise wording.
- [ ] The component avoids jargon unless appropriate.
- [ ] The component supports ETIS doctrine.

## Navigation

- [ ] The component improves location, sequence, or next action.
- [ ] Links are meaningful and specific.
- [ ] The component does not create dead ends.

## Accessibility

- [ ] Color contrast is acceptable.
- [ ] Links are readable.
- [ ] The component works without relying on color alone.
- [ ] The component behaves reasonably on mobile.

## Engineering Fit

- [ ] The component supports Repository-Centered Engineering.
- [ ] The component supports Evidence-Centered Engineering where relevant.
- [ ] The component preserves ETIS tone and identity.
