# Component Priority Register

This register defines the recommended implementation priority.

## Priority 1 — Foundation Components

These should be implemented first.

| Component | Reason |
|---|---|
| Section Hero | Already used on major pages. |
| Navigation Card | Used across section landing pages. |
| Product Button | Needed for clear actions. |
| Next Activity Panel | Improves every instructional page. |
| Callout Base | Foundation for insight, evidence, AI, and governance callouts. |

## Priority 2 — Engineering Stage Components

| Component | Reason |
|---|---|
| Stage Banner | Needed across ES-100 through ES-114. |
| Metadata Strip | Reduces cognitive load. |
| Lifecycle Ribbon | Shows stage position. |
| Evidence Panel | Connects work to evidence. |

## Priority 3 — Readiness Gate Components

| Component | Reason |
|---|---|
| Readiness Gate Header | Gives gates distinct identity. |
| Gate Checklist | Makes readiness criteria reviewable. |
| Gate Decision Panel | Clarifies decision outcomes. |
| Remediation Panel | Supports failed or conditional gates. |

## Priority 4 — Template and Example Components

| Component | Reason |
|---|---|
| Template Purpose Block | Standardizes template pages. |
| Template Usage Panel | Clarifies when to use. |
| Example Guidance Panel | Prevents blind copying. |
| Adaptation Guidance | Helps users apply examples correctly. |

## Priority 5 — Supporting Components

| Component | Reason |
|---|---|
| Related Resources Panel | Improves discoverability. |
| Glossary Link Panel | Improves concept navigation. |
| Progress Tracker | Useful but not required for first migration. |

## Recommendation

Begin with Priority 1 and Priority 2 before redesigning readiness gates.
