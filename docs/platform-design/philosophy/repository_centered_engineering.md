# Repository-Centered Engineering

Repository-Centered Engineering means the repository is the durable center of engineering work.

The repository contains more than code. It contains the artifacts, evidence, decisions, reviews, operating records, governance records, and stewardship history needed to understand and maintain the system.

## Rule

If the work matters, the repository should eventually reflect it.

## Why this matters

Repository-centered practice reduces dependence on private notes, chats, memory, scattered documents, and informal decisions.
