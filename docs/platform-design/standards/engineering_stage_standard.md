# Engineering Stage Standard

Every Engineering Stage must follow the ETIS stage architecture unless a platform-level exception is approved.

## Required stage page set

A standard stage contains:

- `README.md`
- `navigation.md`
- `activities.md` or stage-specific activity pages
- `evidence.md`
- `outputs.md`
- `readiness_gate.md`
- `stage_manifest.md`

ES-100 is an onboarding stage and uses specialized onboarding pages while preserving the same architectural intent.

## Required stage behavior

Each stage must:

- identify its primary engineering question;
- define inputs and outputs;
- identify expected evidence;
- provide clear navigation;
- include a readiness gate;
- include a stage manifest;
- point to the next stage.
