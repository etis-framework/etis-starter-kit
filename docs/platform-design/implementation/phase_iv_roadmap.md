# Phase IV Roadmap — ETIS Platform Design System

## Phase IV-A — Architecture

Deliverables:

- Design System Architecture;
- Design Philosophy;
- Page Taxonomy;
- Component Library;
- Navigation Standards;
- Visual Standards;
- Migration Strategy;
- CSS Architecture;
- Governance Model.

Status: Current package.

## Phase IV-B — Component Specification

Deliverables:

- stage banner spec;
- metadata strip spec;
- readiness gate spec;
- callout spec;
- next activity spec;
- navigation card spec;
- example page spec;
- template page spec.

## Phase IV-C — Pilot Implementation

Recommended pilots:

1. ES-100 Readiness Gate;
2. ES-100 Overview;
3. one Engineering Activity page;
4. one Template page;
5. one LMU/COICP example page.

## Phase IV-D — Page-Type Migration

Migrate by class:

1. landing pages;
2. stage overview pages;
3. readiness gates;
4. engineering activity pages;
5. templates;
6. examples;
7. reference pages.

## Phase IV-E — Freeze

Deliverables:

- build verified;
- CSS stabilized;
- component library frozen;
- migration guide finalized;
- design system versioned.

## Success Criteria

Phase IV succeeds when:

- internal pages feel like the same product as the home page;
- users can always tell where they are;
- page types are visually recognizable;
- navigation is clear;
- readiness gates feel important;
- templates and examples are visually distinct;
- new pages can be created from repeatable templates.
