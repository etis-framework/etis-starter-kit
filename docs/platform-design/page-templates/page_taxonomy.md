# ETIS Page Taxonomy

The ETIS ecosystem should use a limited set of page types. Each page type has a specific purpose and expected structure.

## Page Types

| Type | Purpose |
|---|---|
| Landing Page | Introduces a product or major product area. |
| Section Landing Page | Introduces a major documentation section. |
| Engineering Stage Overview | Introduces an ES stage and its role in the lifecycle. |
| Engineering Activity Page | Guides a specific activity within a stage. |
| Readiness Gate Page | Evaluates whether a stage is ready to close. |
| Template Page | Provides a reusable blank artifact. |
| Example Page | Shows a completed reference artifact. |
| Reference Page | Provides definitions, glossary, policies, or stable reference material. |
| Checklist Page | Provides a structured readiness or review checklist. |
| Index Page | Provides navigation to a set of pages. |

## Required Page Questions

Every page must answer:

1. Where am I?
2. Why am I here?
3. What should I do?
4. What comes next?

## Page Type Summaries

### Landing Page

Used for home pages and top-level product pages.

Required elements:

- hero;
- product identity;
- primary navigation cards;
- audience paths;
- product relationship;
- final call to action.

### Section Landing Page

Used for sections such as Platform, Engineering, Templates, and Examples.

Required elements:

- section hero;
- section purpose;
- navigation cards;
- recommended path;
- next action.

### Engineering Stage Overview

Used for ES-100 through ES-114 overview pages.

Required elements:

- stage banner;
- stage purpose;
- lifecycle position;
- expected outputs;
- readiness logic;
- stage navigation;
- next activity.

### Engineering Activity Page

Used for instructional pages inside a stage.

Required elements:

- page header;
- stage context;
- purpose;
- activity guidance;
- evidence expectation;
- next activity.

### Readiness Gate Page

Used for stage readiness checks.

Required elements:

- readiness gate banner;
- completion standard;
- gate checklist;
- evidence expectations;
- decision outcome;
- next stage or remediation path.

### Template Page

Used for blank reusable artifacts.

Required elements:

- template purpose;
- when to use;
- inputs;
- outputs;
- blank artifact;
- completion guidance.

### Example Page

Used for completed reference artifacts.

Required elements:

- example context;
- artifact purpose;
- what to notice;
- completed example;
- adaptation guidance.

### Reference Page

Used for definitions or lookup material.

Required elements:

- scope;
- stable content;
- cross-links;
- minimal visual overhead.

## Page Taxonomy Rule

A page must declare or clearly imply its type. If a page's type is unclear, the page should be redesigned.
