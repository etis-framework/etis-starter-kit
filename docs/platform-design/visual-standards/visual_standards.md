# ETIS Visual Standards

## Color System

| Token | Use |
|---|---|
| Deep Navy | Background and authority. |
| Navy | Primary brand foundation. |
| Blue | Primary action and navigation. |
| Gold | Emphasis, readiness, doctrine, and ETIS identity. |
| Green | Evidence, completion, successful readiness. |
| Purple | AI, governance, oversight, and model responsibility. |
| White | Content cards and readable panels. |
| Muted Gray | Secondary text. |

## Typography

Principles:

- headings should be confident and readable;
- body text should remain calm and professional;
- long paragraphs should be avoided;
- cards and callouts should use concise text.

## Spacing

Principles:

- landing pages may use larger spacing;
- engineering pages should be denser but still readable;
- readiness gates should feel structured and authoritative;
- reference pages should remain efficient.

## Card Rules

Cards should be used for:

- navigation;
- lifecycle grouping;
- key concepts;
- expected outputs;
- examples;
- page choices.

Cards should not be used for every paragraph.

## Button Rules

Buttons should indicate action.

Examples:

- Begin ES-100;
- Continue to Stage Manifest;
- Open Template Library;
- View Examples.

## Icon Rules

Icons should support meaning and navigation.

Avoid icons that are decorative only.

## Animation Rules

ETIS should use minimal motion:

- hover lift;
- subtle shadow;
- no distracting movement.

## Accessibility

The design system should maintain:

- readable contrast;
- keyboard navigation;
- meaningful link text;
- responsive layouts;
- semantic heading structure.
