## Readiness checklist

- [ ] Planning traces to accepted requirements and constraints.
- [ ] Work packages are deliverable-oriented.
- [ ] Owners are assigned for work, evidence, review, risks, and dependencies.
- [ ] Milestones include readiness evidence.
- [ ] Dependencies are visible and assigned.
- [ ] Estimates include confidence and uncertainty.
- [ ] Risks are recorded, assigned, and actionable.
- [ ] AI verification work is planned where relevant.
- [ ] Architecture-relevant issues are visible.
- [ ] ES-104 can begin without hiding major uncertainty.
