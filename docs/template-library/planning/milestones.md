## Template purpose

Use this template to define milestones for the project.

A milestone should not be a date label only. It should represent a meaningful engineering checkpoint where evidence is reviewed and a decision can be made.

## Review checklist

- [ ] Milestones are tied to evidence, not just dates.
- [ ] Each milestone has an owner.
- [ ] Readiness criteria are defined.
- [ ] Decision options are clear.
- [ ] Milestone risks are identified.
- [ ] Milestones support later architecture, design, implementation, verification, release, and operation.
