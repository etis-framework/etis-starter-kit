## Readiness checklist

- [ ] Problem statement is clear and solution-neutral.
- [ ] Vision responds to the problem and includes trust expectations.
- [ ] Stakeholders include users, affected parties, operators, maintainers, and governance reviewers where relevant.
- [ ] Scope includes in-scope, out-of-scope, deferred, and prohibited items.
- [ ] Material assumptions are visible and assigned.
- [ ] Success criteria include outcome, operational, trustworthiness, and adoption indicators.
- [ ] AI, data, and governance implications are identified where relevant.
- [ ] Requirements work can begin without hiding major uncertainty.
